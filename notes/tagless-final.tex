\emph{Tagless-final encoding} started out as a solution to the \nameref{expression-problem}.
It represents value types as module/trait functions and operations as module/trait instances.

The \code{Shape} example:

\begin{code}[ocaml]
module type Shape = sig
  type t
  val rectangle : float -> float -> t
  val circle : float -> t
end

module Area = struct
  type t = float

  let rectangle w h = w *. h
  let circle r = Float.pi *. r *. r
end

module Perimeter = struct
  type t = float

  let rectangle w h = 2 *. (w +. h)
  let circle r = 2. *. Float.pi *. r
end
\end{code}

Adding new operations is straightforward,
and adding the \code{Triangle} type doesn't require modifying existing code:

\begin{code}[ocaml]
module type ShapeExt = sig
  include Shape
  val triangle : float -> float -> t
end

module AreaExt = struct
  include Area
  let triangle base height = 0.5 *. base *. height
end
\end{code}
